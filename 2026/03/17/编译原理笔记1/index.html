基本概念

​ 这里记录了些零碎的概念，因为我刚开始学习，还未成体系，所以很零碎。

程序本质上是一定字符集上的字符串，语法作为一组规则，产生合式的程序

词法规则

定义：单词符号的形成规则，一般包括常数、标识符、基本字、算符、界符等

描述工具：有限自动机

语法规则

定义：语法单位的形成规则，通常包括表达式、语句、分程序、过程、程序、函数等

描述工具：上下文无关文法

上下文无关文法

1. 基本定义

上下文无关文法是一个四元组：G = (Vn， Vt， S， P)

其中：

  • Vn：非终结符集合

    • 代表语法成分，如〈表达式〉、〈语句〉等
    • 通常用尖括号〈〉表示，或用大写字母A、B、C等表示

  • Vt：终结符集合

    • 组成语言的基本符号，是词法分析的结果
    • 通常用小写字母a、b、c或具体符号如+、-、if等表示

  • S：开始符号（S ∈ Vn）

    • 语言中最大的语法单位
    • 所有推导的起点

  • P：产生式集合

    • 形如：A → β，其中 A ∈ Vn， β ∈ (Vn∪Vt)

2. 相关概念

（1）直接推导与推导

  • 直接推导：若 A → γ ∈ P，则 αAβ ⇒ αγβ（用→的右部替换左部）
  • 推导：α ⇒* β 表示从α出发，经过零步或多步直接推导得到β

（2）句型与句子

  • 句型：从开始符号S出发，推导得到的任意符号串（可含非终结符）
  • 句子：从开始符号S出发，推导得到的只含终结符的符号串
  • 语言：所有句子的集合，记作 L(G) = { ω | S ⇒ ω， ω ∈ V_T }

（3）最左推导与最右推导

  • 最左推导：每次都替换最左边的非终结符
  • 最右推导：每次都替换最右边的非终结符（也称规范推导）

3. 语法分析树

定义：用树形结构表示推导过程的图形工具

性质：

  • 根节点：开始符号S
  • 内部节点：非终结符
  • 叶子节点：终结符或ε
  • 每个内部节点与其子节点对应一个产生式
  • 最左推导与最右推导对应同一棵语法分析树

作用：直观展示句子的语法结构，是语义分析的基础

4. 二义性（Ambiguity）

定义：若一个句子存在两棵或以上不同的语法分析树（或等价地，存在两个不同的最左/最右推导），则该文法是二义的

例子：文法 E → E + E | E E | id 对于句子 id + id id 存在两种推导

处理方法：

  • 改写文法：引入优先级和结合性规则
    • E → E + T | T
    • T → T * F | F
    • F → (E) | id
  • 规定优先级和结合性：在分析器中明确约定

5. 乔姆斯基范式（Chomsky Normal Form，CNF）

任何上下文无关文法（不含ε产生式）都可以转化为如下形式：

  • A → BC（两个非终结符）
  • A → a（一个终结符）

用途：

  • CYK算法需要CNF
  • 理论分析方便

6. 上下文无关文法的能力与局限

能力：

  • 描述大多数程序设计语言的语法结构
  • 能处理嵌套结构（如括号匹配、if嵌套）

局限：

  • 不能描述：变量先声明后使用、类型匹配等上下文相关的要求（这些留给语义分析处理）

7. 与词法分析的关系

比较维度 词法规则 语法规则
描述对象 单词符号 语法单位
描述工具 正则文法/有限自动机 上下文无关文法/下推自动机
处理能力 线性结构 嵌套结构
识别器 有限自动机 下推自动机
